# Day 1 AM — Mechanism Menu

This morning is deliberately overfull. Do not try to finish every exercise.

The exercises support a mixed room:

- beginners can build their first Skill
- intermediate participants can practice planning artifacts
- advanced participants can explore Rules, Subagents, and stricter factory ingredients

## Shared Kickoff

Use the first 10 minutes to explain:

- `CLAUDE.md` is always-on context
- Skills are on-demand workflows
- Rules are scoped by file glob
- Subagents are isolated delegated contexts
- Hooks are deterministic tool-event guardrails
- factory stations communicate through artifacts and contracts

Then split participants by readiness.

## Track A — Starter / Intermediate

Recommended order:

1. [00 Pirate CLAUDE.md](00-pirate-claude-md/README.md)
2. [01 Pirate Skill](01-pirate-skill/README.md)
3. [04 Planning Skill](04-planning-skill/README.md)

This track is for people who have never created a Skill before or need low-friction exercises while tooling is being resolved.

## Track B — Advanced

Recommended order:

1. [02 Rules](02-rules/README.md)
2. [03 Subagent](03-subagent/README.md)
3. [04 Planning Skill](04-planning-skill/README.md), but make it stricter

This track is for people who already know Skills and want mechanisms that become useful inside a factory.

## Convergence Point

The planning Skill is the morning convergence point, but nobody needs to finish it for the afternoon to work.

Afternoon fallback options:

- use the supplied planner
- use a pre-seeded Bean
- pair with someone who completed a planning Skill

The real dependency is not "everyone built the same thing." The real dependency is:

```text
Afternoon needs a Bean with ## High-Level Plan.
```
